# 프로젝트 컨텍스트: 하이브리드 오디오 전략 고도화 (Librosa + 2단계 AI 분석)

## 1. 개요
LLM의 기술적 오디오 분석 한계(Hallucination)를 인정하고, **'수학적 정밀도(Librosa)'**와 **'AI의 감성적 선택'**을 결합한 2단계 하이브리드 파이프라인을 구축함. 음악의 박자가 엇나가는 문제와 볼륨 불균형을 원천적으로 해결하는 것이 목표.

## 2. 작업 완료 및 검증 사항

### 2.1 지능형 음성 배속 보정 (Voice Duration Scaling) 구현 완료
- **핵심 성과**: `32.43s` -> `30.00s` 정밀 압축 및 자막 싱크 보정 로직 안착.
- **기술 스택**: Replicate `ffmpeg-sandbox-2` 활용.

### 2.2 음악 분석 실패 분석 및 피벗 결정
- **한계 발견**: LLM이 긴 오디오에서 정밀한 `startSec`을 찍지 못하는 문제 확인.
- **해결책**: AI에게 주관식(계산)이 아닌 객관식(비교) 문제를 내는 방식으로 구조 변경.

## 3. 확정된 전략: Librosa 기반 2단계 하이브리드 파이프라인
'박자 미스'와 '볼륨 불균형'을 근본적으로 해결하기 위한 구조.

### 3.1 파이프라인 상세 단계
1.  **[AI 1차] 곡 선택 (Song Selection)**: 2개의 후보 곡 중 비디오 주제와 분위기에 더 잘 어울리는 1곡을 AI가 먼저 낙점.
2.  **[Librosa] 하이라이트 후보 추출**: 선택된 곡을 Librosa로 분석하여 에너지 피크 지점을 찾고, 가장 가까운 **'다운비트(Downbeat)'** 지점부터 30초짜리 후보 클립 3개를 생성. (수학적으로 박자를 맞춤)
3.  **[AI 2차] 최종 매칭 및 볼륨 결정**: 나레이션 오디오와 3개의 후보 클립을 비교하여 가장 감정선이 잘 맞는 하나를 선택하고, 나레이션 대비 최종 믹싱 볼륨(Weight)을 결정.
4.  **[FFmpeg] 정밀 병합**: `loudnorm`으로 나레이션과 음악의 볼륨을 표준화한 뒤, AI가 준 가중치를 적용하여 기계적으로 정렬/병합.

## 4. 향후 작업 (Next Steps) - [Priority: HIGH]
1.  **Librosa 기반 Replicate 모델 배포**: Python/Librosa를 사용하여 비트에 맞춰 하이라이트를 추출하는 스크립트 작성 및 모델 업로드.
2.  **`llmServerAPI.ts` 분리 및 개조**: `postMusicAnalysis`를 1차(곡 선택)와 2차(최종 매칭) 함수로 분리하고 프롬프트 전면 개편.
3.  **`/api/autopilot/music` 로직 개편**: 2단계 AI 호출과 중간 Librosa 분석 과정을 순차적으로 연결하는 파이프라인 구축.
4.  **볼륨 밸런스 로직 구현**: `loudnorm` 적용 후 AI 가중치를 반영하여 Suno/ElevenLabs 간의 볼륨 편차를 해결하는 최종 병합 로직 완성.
