# 프로젝트 컨텍스트: 오토파일럿 전용 OAuth 및 UI 정렬 최적화

## 1. 개요
오토파일럿 설정 창에서 플랫폼 계정을 연동할 수 있도록 OAuth 진입점을 분리하고, 사이드바 UI 디자인에 맞춰 연동 버튼의 시각적 정렬 및 디자인 일관성을 확보하는 작업을 완료했습니다.

## 2. 주요 결정 사항 및 구현 완료 사항
- **오토파일럿 전용 OAuth 엔드포인트 구축 (완료)**:
    - `youtube/autopilot/oauth` 및 `tiktok/autopilot/oauth`: `seriesId`를 파라미터로 수용하고, `mode: 'autopilot'` 상태를 보존하여 콜백에 전달하도록 구현.
- **통합 UI 컴포넌트 구축 및 정렬 (완료)**:
    - `PlatformConnectButton.tsx` & `PlatformCheckbox.tsx`: 연동 전/후의 UI를 책임지는 통합 컴포넌트 제작.
    - **X축 정렬 완성**: 아이콘 영역에 `mr-8` 마진을 고정하여 모든 플랫폼의 텍스트 시작점을 완벽하게 일치시킴.
    - **디자인 조화**: `gray-800` 배경과 보라색 테두리(`border-purple-500/30`)를 적용하여 다크 모드 사이드바와 조화를 이루도록 설계.
- **실제 계정 연동 상태 UI 반영 (완료)**:
    - `autopilotDataClientAPI.getAutopilotDataPlatformConnection()`을 활용하여 실제 OAuth 토큰 유무에 따라 버튼/체크박스가 동적으로 전환되도록 구현.
    - 로딩 중 스피너(`Loader2`) 처리로 사용자 경험 개선.
- **미완성 플랫폼 대응 (완료)**:
    - 인스타그램 등 미지원 플랫폼에 대해 렌치(Wrench) 아이콘 및 `isProgressing` 상태를 적용하여 시각적 일관성 유지.

## 3. 현재 상태 (Status)
- **UI 및 데이터 연동 완성**: 실제 토큰 상태에 따른 UI 스위칭과 픽셀 단위의 정렬 작업이 모두 완료되었습니다.
- **코드 리팩토링**: `AutopilotControlPanel.tsx`에서 지저분한 스타일 오버라이드를 제거하고 선언적인 삼항 연산자 구조로 개선했습니다.

## 4. 향후 작업 및 기술적 메모 (Next Steps)
1.  **콜백(Callback) 리다이렉션 검증**:
    *   OAuth 인증 성공 후 설정 페이지로 돌아왔을 때, 캐시된 데이터가 아닌 최신 연동 상태가 즉시 반영되는지 최종 확인.
2.  **유튜브 공개 범위 UI**:
    *   유튜브 연동 성공 시 나타나는 체크박스 하단에 `Privacy Setting` 드롭다운 추가.
3.  **연동 해제(Disconnect) 기능**:
    *   연동된 플랫폼을 다시 해제하거나 계정을 변경할 수 있는 UI 및 백엔드 로직 검토.
