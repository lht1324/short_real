# 프로젝트 컨텍스트: 하이브리드 오디오 전략 고도화 (멀티 클립 후보 방식)

## 1. 개요
LLM의 기술적 오디오 분석 한계를 인정하고, AI의 '감성적 비교 능력'과 서버의 '수학적 정밀도'를 결합한 새로운 오디오 파이프라인 전략을 수립함. 기존의 'AI 통합 분석' 방식에서 '서버 후보 추출 + AI 최적 선택' 방식으로 피벗함.

## 2. 작업 완료 및 검증 사항

### 2.1 지능형 음성 배속 보정 (Voice Duration Scaling) 구현 완료
- **핵심 성과**: `32.43s` -> `30.00s` 정밀 압축 및 자막 싱크 보정 로직 안착.
- **기술 스택**: Replicate `ffmpeg-sandbox-2` 활용.

### 2.2 음악 분석 실패 분석 및 피벗 결정
- **한계 발견**: Gemini 3.1 Pro/Flash 등 현존 LLM은 긴 오디오 파일에서 정밀한 `startSec` 계산이나 볼륨 % 산출 시 '찍기' 현상(Hallucination) 발생.
- **해결책**: AI에게 주관식(계산)이 아닌 객관식(비교) 문제를 내는 방식으로 구조 변경.

## 3. 확정된 전략: 3단계 멀티 클립 파이프라인
'박자 미스'와 '볼륨 불균형'을 근본적으로 해결하기 위한 하이브리드 구조.

### 3.1 파이프라인 단계
1.  **후보 추출 (Replicate/Server)**: 
    - 음악 파일 전체(3분+)를 분석하여 에너지 피크(Onset/Peak)가 높은 지점 3~5곳을 추출.
    - 각 지점부터 숏폼 길이(30초)로 잘라낸 클립들을 생성. (이미 비트에 맞춰 잘려 있으므로 박자 미스 방지)
2.  **최적 선택 (Gemini 3.1 Flash)**:
    - 잘라진 클립 후보군과 나레이션을 동시에 입력받아, 가장 어울리는 클립 인덱스와 믹싱 모드(Quiet/Normal/Intense)를 선택.
3.  **수학적 병합 (FFmpeg/Replicate)**:
    - `loudnorm` (EBU R128) 필터를 사용하여 나레이션과 음악의 인지적 음량을 동일하게 맞춤.
    - AI가 선택한 모드에 따른 고정 배수(예: 0.25x)를 적용하여 기계적으로 믹싱.

## 4. 향후 작업 (Next Steps) - [Priority: HIGH]
1.  **Replicate 전처리 스크립트 작성**: 음악에서 하이라이트 후보 구간을 자동 추출하는 Python/FFmpeg 로직 구현.
2.  **프롬프트 및 API 개조 (`POST_MUSIC_ANALYSIS`)**: 통파일 분석이 아닌 멀티 클립 비교 방식으로 `llmServerAPI.ts` 및 프롬프트 수정.
3.  **믹싱 모드 프리셋 정의**: `loudnorm` 적용 후 각 모드별로 곱할 볼륨 상수값(Weight) 확정.
4.  **전체 파이프라인 통합 테스트**: 새로운 하이브리드 방식으로 제작된 영상의 오디오 품질 최종 검증.
