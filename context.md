# 프로젝트 컨텍스트: 오토파일럿 전용 OAuth 및 UI 정렬 최적화

## 1. 개요
오토파일럿 설정 창에서 플랫폼 계정을 연동할 수 있도록 OAuth 진입점을 분리하고, 인증 완료 후 정상적으로 설정 페이지로 복귀하는 하이브리드 콜백 시스템을 구축했습니다. 또한 사이드바 UI를 통합 컴포넌트로 리팩토링하여 디자인 일관성을 확보했습니다.

## 2. 주요 결정 사항 및 구현 완료 사항
- **하이브리드 콜백 시스템 구축 (완료)**:
    - `callback/youtube` & `callback/tiktok`: `mode === 'autopilot'` 여부에 따라 로직 분기.
    - **오토파일럿 모드**: `taskId` 없이 `seriesId`만으로 동작하며, 연동 성공 시 `/workspace/autopilot`으로 리다이렉트.
    - **매뉴얼 모드**: 기존 수동 업로드 로직(`taskId` 필수, 대시보드 리다이렉트)을 100% 유지하여 하위 호환성 확보.
- **OAuth UX 지능화 (완료)**:
    - `youtube/autopilot/oauth` & `tiktok/autopilot/oauth`: 이미 유효한 토큰이 있는 경우, 인증 창을 띄우지 않고 즉시 설정 페이지로 리다이렉트하도록 개선.
- **통합 UI 컴포넌트 및 정렬 (완료)**:
    - `PlatformConnectButton.tsx` & `PlatformCheckbox.tsx`: 연동 전/후 UI 통합.
    - **X축 정렬**: `mr-8` 고정을 통해 모든 플랫폼의 텍스트 시작점 일치.
    - **다크 모드 최적화**: `gray-800` 배경과 보라색 테두리 포인트 적용.
- **실제 계정 연동 상태 UI 반영 (완료)**:
    - `autopilotDataClientAPI.getAutopilotDataPlatformConnection()`을 활용하여 실제 OAuth 토큰 상태와 UI를 동기화.
- **미완성 플랫폼 대응 (완료)**:
    - 인스타그램 등 미지원 플랫폼에 렌치(Wrench) 아이콘 및 `isProgressing` 상태 적용.

## 3. 현재 상태 (Status)
- **End-to-End 연동 완성**: 설정 창에서 'Connect' 클릭부터 인증 후 복귀까지의 전체 흐름이 오토파일럿 모드에 맞게 완성되었습니다.
- **하위 호환성 유지**: 기존 수동 업로드 기능에 영향을 주지 않도록 하이브리드 패턴으로 구현되었습니다.

## 4. 향후 작업 및 기술적 메모 (Next Steps)
