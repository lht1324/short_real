# 프로젝트 컨텍스트: 오토파일럿 파이프라인 최적화 및 음성 보정 설계

## 1. 개요
오토파일럿 시스템의 안정적인 가동을 위해 스케줄러(Orchestrator)와 메타데이터 생성 API 간의 연동을 최적화하고, 제작 비용 절감 및 품질 유지를 위한 '지능형 음성 배속 보정' 로직을 설계했습니다.

## 2. 지능형 음성 배속 보정 (Voice Duration Scaling) 설계
비용($0.06/1초)과 품질(자연스러운 발화) 사이의 최적 지점을 찾는 하이브리드 보정 모델입니다.

### 2.1 설계 원칙
- **비용 최적화**: 30.0초를 목표(Target)로 하여 지출을 최소화함.
- **품질 마지노선**: **최대 1.15배속** (Scaling Ratio 0.869) 리미트 적용.
  - **감량 한계치**: 타임스탬프 기준 원본 길이의 **최대 약 13%($1 - 1/1.15$)**까지만 줄일 수 있음. 이를 초과하여 30초에 맞추려 하지 않음으로써 '랩 현상'을 방지함.
- **조건부 보정**: 32.0초 이하의 영상은 원본 품질 보존을 위해 보정을 적용하지 않음 (Ignore Threshold).

### 2.2 알고리즘 (Ratio-Based Scaling Model)
1. **입력**: 원본 음성 전체 길이 ($T_{orig}$)
2. **보정 가능 최대치 계산**: $T_{limit} = 30.0 \times 1.15 = 34.5s$
3. **조건부 목표 길이 ($T_{target}$)** 산출:
   - **Case A ($T_{orig} \le 32s$)**: 보정 없음 ($T_{target} = T_{orig}$)
   - **Case B ($32s < T_{orig} \le 34.5s$)**: 완벽한 30초행 ($T_{target} = 30.0s$)
   - **Case C ($T_{orig} > 34.5s$)**: 최대 배속(1.15x) 적용 ($T_{target} = T_{orig} / 1.15$)
4. **계산**: 배속 계수 $R = T_{target} / T_{orig}$
5. **적용**: 모든 자막 타임스탬프 및 최종 메타데이터에 계수 $R$을 곱함.

| **짧은 영상** | 25.0s | 25.0s | 1.0x (무보정) | 보정 불필요, 비용 $1.50 |

## 3. 오케스트레이터 및 API 연동 최적화
- **시간 데이터 소유권**: `last_run_at`은 오케스트레이터가 비즈니스 날짜 기준으로 관리하며, API 엔드포인트는 이를 오염시키지 않도록 수정함.
- **테스트 관측성**: `video-metadata` API 내부에 구간별 테스트 포인트를 배치하여 디버깅 효율 증대.
- **Fire & Forget 안정화**: 내부 호출 시 `x-internal-secret` 헤더 자동 포함 및 `waitUntil`을 통한 비동기 처리 보장.
